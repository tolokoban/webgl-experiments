Dans ce chapitre, nous allons utiliser WebGL pour afficher 150 sprites. Vous pouvez constater que ça reste fluide. Pour WebGL, chaque sprite sera un carré formé de 4 vertices qui auront tous les mêmes coordonnées (celles du centre du sprite) ainsi que les attributs suivants : * Index du vertex (de 0 à 3). Les points se suivent dans le sens inverse des aiguilles d'une montre. * Angle de rotation en radians. * Facteur de zoom. Si ce facteur vaut 7, par exemple, le sprite fera 14 x 14 pixels. Nous aurons donc besoin des attributs suivants : attribute vec3 attPosition; attribute float attIndex; attribute float attRotation; attribute float attScale; Le _fragment shader_ est relativement simple : Côté Javascript, nous allons faire varier les valeurs des attributs de chaque sprite en fonction du temps. Ainsi, à chaque frame, nous devrons rafraichir le buffer lié à ces attributs. Nous allons maintenant définir le lien entre un buffer et les attributs dont on a besoin. Notre vertex shader aura besoin des 4 attributs suivants : * `attribute vec3 attPosition; // offset 0` * `attribute float attIndex; // offset 3` * `attribute float attScale; // offset 4` * `attribute float attRotation; // offset 5` Il nous faut donc 6 `float` pour définir un vertex : c'est la taille d'un _bloc_. A l'appel de __drawArrays()__, WebGL va lire toutes les données d'un bloc et alimenter les attributs avec ces valeurs avant d'appeler le premier microprocesseur de vertex shader disponible. # Les textures à partir d'images
